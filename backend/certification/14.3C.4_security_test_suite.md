## Test Files

### 1. `tests/security/test_tenant_rls.py` — Tenant Isolation Tests

| Test | What It Verifies |
|------|-----------------|
| `test_orchestrator_rejects_cross_tenant_goal` | `orchestrator.get_goal_status()` raises `ValueError` for cross-tenant goal_id |
| `test_agent_registry_rejects_cross_tenant` | `agent_registry.get_agent()` returns `None` for cross-tenant agent_id |
| `test_approval_service_rejects_cross_tenant` | `approval_service.process_approval()` raises `ValueError` for cross-tenant approval_id |
| `test_action_registry_rejects_cross_tenant` | `action_registry.get_action()` returns `None` for cross-tenant action_id |
| `test_plan_endpoint_cross_tenant_404` | HTTP 404 when tenant B tries to GET a plan owned by tenant A |
| `test_goal_endpoint_cross_tenant_404` | Cross-tenant goal access denied |

**Methodology:** FakeSession stores entities for tenant A. All lookups use tenant B. Service-layer tests expect `ValueError` or `None`. Endpoint tests expect HTTP 404.
