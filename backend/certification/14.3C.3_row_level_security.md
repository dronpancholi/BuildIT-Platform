## Policy Design

### Mechanism
- RLS policies use `current_setting('app.current_tenant')::uuid` to match against `tenant_id`
- This session variable is set by `get_tenant_session()` in `core/database.py:201`
- Four per-statement policies per table:
  - `FOR SELECT` — `USING (tenant_id = current_setting(...))`
  - `FOR INSERT` — `WITH CHECK (tenant_id = current_setting(...))`
  - `FOR UPDATE` — `USING (...)` + `WITH CHECK (...)` (prevents changing tenant_id)
  - `FOR DELETE` — `USING (tenant_id = current_setting(...))`
