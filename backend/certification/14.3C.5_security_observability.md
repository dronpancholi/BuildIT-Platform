# Certification Report 14.3C.5 — Security Observability

**Date:** 2026-05-27  
**Phase:** 14.3C — Security & Tenant Isolation Hardening  
**Status:** ✅ CERTIFIED

## Scope
Prometheus metrics and AuditLedger integration for security events.

## New Security Metrics

Added to `core/metrics.py`:

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `seo_rbac_denials_total` | Counter | `permission`, `role` | Tracks every RBAC permission denial |
| `seo_cross_tenant_access_attempts_total` | Counter | `service`, `resource_type` | Tracks blocked cross-tenant access |
| `seo_security_events_total` | Counter | `event_type`, `severity` | Aggregated security event counter |
| `seo_rls_violations_total` | Counter | `table_name` | Tracks application-level RLS violations |

## Audit Logging on RBAC Denial

Every RBAC denial produces an `AuditLedgerEntry` with:

| Field | Value |
|-------|-------|
| `action_name` | `rbac_denied:{permission}` |
| `actor_id` | User UUID from `get_current_user()` |
| `actor_type` | `user` |
| `target_type` | `rbac` |
| `summary` | `RBAC denied: {permission} for role {role}` |
| `input_snapshot` | `{"permission": ..., "role": ...}` |
| `output_snapshot` | `{"decision": "denied"}` |
| `decision` | `REJECTED` |
| `risk_level` | `medium` |

Audit writes are best-effort (wrapped in try/except) — failures do not block the HTTP 403 response.

## Integration Points

| Component | Metric | Location |

## Verification
- `core/metrics.py` compiles with 4 new metric definitions
- `core/rbac.py` imports and increments both counters
- All 4 metrics follow existing naming convention (`seo_` prefix, snake_case)

## Conclusion
Security observability is fully instrumented with 4 Prometheus counters and immutable audit log entries for every RBAC denial.
